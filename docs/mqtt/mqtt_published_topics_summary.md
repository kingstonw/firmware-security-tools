# MQTT Published Topics Summary

## Scope

This report covers confirmed code paths for these production published topics:

- `fc75/tx/system/error`
- `fc75/tx/response`
- `fc75/tx/system/ota`
- `fc75/tx/system/log`

Only behavior confirmed in executable code is included.

## Topic Publish Conditions (Confirmed)

- `fc75/tx/system/error`
  - Published from telemetry `ERROR` slot.
  - Requires telemetry scheduler execution with Wi-Fi connected and no button pressed.
  - Requires queued records in `/error.log`.

- `fc75/tx/system/ota`
  - Published from telemetry `OTA` slot.
  - Requires telemetry scheduler execution with Wi-Fi connected and no button pressed.
  - Requires queued records in `/ota.log`.

- `fc75/tx/system/log`
  - Published from telemetry `LOG` slot.
  - Requires telemetry scheduler execution with Wi-Fi connected and no button pressed.
  - Generates a fresh system log record, then sends queued `/log.log` records.

- `fc75/tx/response`
  - Published when command handlers call `AWS_sendCommandResponse()`.
  - Triggered in two places:
  - Immediate responses in `aws_defaultRemoteCommandCallback()`.
  - Deferred responses in `TRIGGERS_process()` (for example `STM_RESULT`).

## Related Functions and File Locations

- `AWS::AWS_sendEvents()` in [AWS.cpp](AWS.cpp)
  - Builds JSON with fields `mac`, `type`, `timestamp`, `mode`, `events` and publishes to caller-selected topic.

- `AWS::AWS_sendCommandResponse()` in [AWS.cpp](AWS.cpp)
  - Builds JSON with fields `mac`, `type`, `timestamp`, `command`, `mode`, `status` and publishes to `AWS_RESPONSE` (`fc75/tx/response`).

- `TELEMETRY_scheduler(bool sendOnline)` in [main.ino](main.ino)
  - Chooses `ERROR`, `OTA`, `LOG` publish paths and target topics (`DEV_*` vs `AWS_*`).

- `RECORD_sendAWS(...)` in [main.ino](main.ino)
  - Reads records from SPIFFS logs, joins records with `~`, calls `AWS_sendEvents()`, and deletes sent records on success.

- `aws_defaultRemoteCommandCallback(...)` in [AWS.cpp](AWS.cpp)
  - Handles inbound MQTT commands and emits immediate command responses.

- `TRIGGERS_process()` in [main.ino](main.ino)
  - Executes deferred command actions and emits deferred command responses (for example `STM_RESULT`).

## Additional Notes

- `DEV_RESPONSE` is defined in config, but `AWS_sendCommandResponse()` currently publishes unconditionally to `AWS_RESPONSE` (`fc75/tx/response`).
- Timings for telemetry slots are configured in `config::DEVICE`:
  - `ERROR_SEND_INTERVAL_MS`
  - `OTA_SEND_INTERVAL_MS`
  - `LOG_SEND_INTERVAL_MS`
- Field optionality for these published payloads is `not confirmed in code`; all listed fields are explicitly set before publish.

## Real SYSTEM_LOG Example and Events Field Breakdown

### Example Payload (Observed)

```json
{
  "mac": "30:ED:A0:A7:B8:24",
  "type": "SYSTEM_LOG",
  "timestamp": "2026-04-01T17:51:37Z",
  "mode": "PROD",
  "events": "2026-04-01T17:51:37Z|1.0.63|161724|449792|9570881|-46|4,16,1,10|0"
}
```

### How to Parse `events`

`events` is a pipe-delimited (`|`) record. In this project, a SYSTEM_LOG record body is built in `RECORD_assembler(LOG)`, then Logger writes it to SPIFFS with a leading timestamp prefix. So the parsed order is:

| Position | Example Value | Meaning |
| --- | --- | --- |
| 1 | `2026-04-01T17:51:37Z` | Record timestamp prefix added by logger write path. |
| 2 | `1.0.63` | ESP32 firmware version (`config::DEVICE::VERSION`). |
| 3 | `161724` | Free heap bytes (`ESP.getFreeHeap()`). |
| 4 | `449792` | SPIFFS used bytes (`SPIFFS.usedBytes()`). |
| 5 | `9570881` | SPIFFS total bytes (`SPIFFS.totalBytes()`). |
| 6 | `-46` | Wi-Fi RSSI in dBm (`WiFi.RSSI()`). |
| 7 | `4,16,1,10` | Uptime as `days,hours,minutes,seconds` from `UTIL_formatTime()`. |
| 8 | `0` | Unsent error record count in queue (`SPIFF_getTotalRecords(ERRORLOG)`). |

### Notes on Dual Timestamps

- Top-level JSON `timestamp` is the MQTT message timestamp field.
- The first value inside `events` is the log-record timestamp stored in SPIFFS.
- They can be identical or near-identical depending on when the queued record is created and flushed.
